Clasp wiring + push UI removal

Six decisions to wire up @google/clasp so the Apps Script for the Bonds Ranch schedule sheet lives in the repo, and to strip the "Setup push" affordance from the toolbar. Some choices cascade — Q4 depends on Q1, and Q2 needs a Script ID if you've already deployed the script once. Every card has a 🔍 Sub-agent deep dive option and a free-text escape hatch.

docs/decisions/decision-sheet-2026-05-23-clasp-push-ui.html · 2026-05-23

Q1How far should the push-to-sheet UI come out?

"Setup push" is the gear-icon button shown when no webhook URL is saved. It opens the 3-step Apps Script deploy panel. The button morphs into "Push to Sheet" once a URL is saved. You said you don't want "Setup push" in the UI — this question pins down how much of the push apparatus goes with it.

Trade-off: If the entire push UI goes, the React app can no longer write back to the sheet — the sheet stays source of truth and the app is pull-only. Clasp pushes the script code to Apps Script, but doesn't sync row data. You'd edit the sheet directly (or via the Apps Script editor) to change the schedule. The "Sync Sheet" pull button stays either way.

Q2Apps Script project — does one already exist for this sheet?

Clasp needs to know whether to bind to an existing Apps Script project or create a fresh one parented to the sheet.

How to tell: open the sheet → Extensions → Apps Script. If you see code already there (the doPost function or anything else), a project exists — copy its Script ID from the editor URL (the long string between /projects/ and /edit) and paste below. If the editor is empty/never opened, pick "Create new".

Q3Where should the GAS source live in the repo?

Where to drop Schedule.gs + appsscript.json (the clasp-managed sources). Convention only — clasp doesn't care.

Q4What to do with the inline GAS_SCRIPT const in App.tsx?

App.tsx:40–66 holds the Apps Script source as a template literal — its only consumer is the "Copy Script" button in the panel. If the panel goes (Q1=remove all), this becomes dead code. If the panel stays in any form, the const matters.

Cascade: If you picked "Remove all of it" on Q1, the safe default is delete. If you kept any push UI, pick the Vite ?raw import option — that way the inline string and the clasp-managed .gs file can never drift.

Q5How should clasp be installed?

Project uses pnpm. Clasp can be a repo devDep, global install, or run on-demand via pnpm dlx.

Q6After scaffolding — drive the OAuth + initial push myself, or hand off?

Either I (a) print exact commands and stop, or (b) actually run clasp login, clasp create/clone, and clasp push from this terminal.

If "Drive it": a browser OAuth window will pop up on this machine — you'll need to be at the keyboard to grant access. Apps Script API also has to be enabled at script.google.com/home/usersettings before clasp create works.
